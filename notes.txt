# So, the @dataclass decorator is not mandatory, but it saves you from boilerplate code when your class is primarily used to store data.
You have to manually define __init__, __repr__, and sometimes __eq__.
With @dataclass, Python auto-generates those for you: so we can simply go ahead with defining the variables with no initialisation.
